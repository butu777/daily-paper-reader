## 摘要
从视频预测未来状态的世界模型仍受限于扁平的潜表示，这些表示将物体纠缠在一起，忽略了因果结构，并将时间动力学压缩到单一尺度。我们提出了 HCLSM，这是一种基于三个相互关联原则运行的世界模型架构：通过带有空间广播解码的槽注意力（slot attention）进行以物体为中心的分解；通过三层引擎实现分层时间动力学，该引擎结合了用于连续物理的选择性状态空间模型（SSM）、用于离散事件的稀疏 Transformer 以及用于抽象目标的压缩 Transformer；以及通过图神经网络交互模式进行因果结构学习。HCLSM 引入了一种两阶段训练协议，其中空间重构在动力学预测开始前强制槽特化（slot specialization）。我们在 Open X-Embodiment 数据集的 PushT 机器人操控基准上训练了一个拥有 6800 万参数的模型，实现了 0.008 MSE 的下一状态预测损失，并展现出涌现的空间分解（SBD 损失：0.0075）和学习到的事件边界。用于 SSM 扫描的自定义 Triton 内核相比顺序 PyTorch 实现了 38 倍的加速。整个系统包含分布在 51 个模块中的 8,478 行 Python 代码，并配有 171 个单元测试。代码：https://github.com/rightnow-ai/hclsm

## Abstract
World models that predict future states from video remain limited by flat latent representations that entangle objects, ignore causal structure, and collapse temporal dynamics into a single scale. We present HCLSM, a world model architecture that operates on three interconnected principles: object-centric decomposition via slot attention with spatial broadcast decoding, hierarchical temporal dynamics through a three-level engine combining selective state space models for continuous physics, sparse transformers for discrete events, and compressed transformers for abstract goals, and causal structure learning through graph neural network interaction patterns. HCLSM introduces a two-stage training protocol where spatial reconstruction forces slot specialization before dynamics prediction begins. We train a 68M-parameter model on the PushT robotic manipulation benchmark from the Open X-Embodiment dataset, achieving 0.008 MSE next-state prediction loss with emerging spatial decomposition (SBD loss: 0.0075) and learned event boundaries. A custom Triton kernel for the SSM scan delivers 38x speedup over sequential PyTorch. The full system spans 8,478 lines of Python across 51 modules with 171 unit tests. Code: https://github.com/rightnow-ai/hclsm

当前的世界模型（如 V-JEPA、DreamerV3）在处理视频预测时存在三大局限：
*   **物体纠缠（Entanglement）：** 采用扁平的潜空间（Flat Latent Spaces），无法将场景中的独立实体（如机器人抓手、杯子、桌面）分开表示。
*   **时间尺度单一：** 物理世界的动态包含连续运动（毫秒级）、离散事件（如碰撞）和长期目标（分钟级），单一尺度的 Transformer 难以同时捕捉这些特征。
*   **缺乏因果结构：** 模型无法理解“谁影响了谁”，导致难以进行反事实推理（如“如果推力加大，物体会如何移动”）。
**HCLSM 的目标**是构建一个符合物理世界结构的架构，通过物体中心化分解、分层动力学和因果图学习，使模型具备更强的推理和规划能力。

### 2. 方法论：核心思想与关键技术
HCLSM 提出了一个五层架构，并配合独特的“两阶段训练协议”：

*   **核心架构层次：**
    1.  **感知层：** 使用 ViT 编码器将视频帧转化为 Patch 嵌入。
    2.  **物体分解层：** 利用 **Slot Attention（槽注意力）** 将场景分解为多个 Slot。引入 **空间广播解码器（SBD）**，通过重构 ViT 特征（而非像素）来强制 Slot 学习特定空间区域。
    3.  **分层动力学引擎：**
        *   **L0（连续物理）：** 为每个 Slot 分配独立的 **选择性状态空间模型（SSM/Mamba）**，处理帧间连续运动。
        *   **L1（离散事件）：** 稀疏 Transformer，仅在检测到“事件边界”（状态突变）时触发。
        *   **L2（抽象目标）：** 压缩 Transformer，将事件序列总结为高层目标。
    4.  **因果推理层：** 通过图神经网络（GNN）学习 Slot 间的交互，利用 NOTEARS 约束学习有向无环图（DAG）结构。
    5.  **持续记忆层：** 结合 Hopfield 网络和 EWC 算法处理长期记忆。

*   **关键算法流程：两阶段训练（Two-Stage Training）**
    *   **阶段 1（前 40% 步数）：** 仅激活 SBD 重构损失。强制 Slot 竞争空间所有权，完成“物体发现”。
    *   **阶段 2（后 60% 步数）：** 激活 JEPA 风格的动力学预测损失。在已解耦的物体基础上学习它们如何随时间演化。
