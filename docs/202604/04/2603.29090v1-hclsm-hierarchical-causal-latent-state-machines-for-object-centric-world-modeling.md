## 摘要
从视频预测未来状态的世界模型仍受限于扁平的潜表示，这些表示将物体纠缠在一起，忽略了因果结构，并将时间动力学压缩到单一尺度。我们提出了 HCLSM，这是一种基于三个互连原则运行的世界模型架构：通过带有空间广播解码的槽注意力（slot attention）进行以物体为中心的分解；通过结合了用于连续物理的选择性状态空间模型（SSM）、用于离散事件的稀疏 Transformer 以及用于抽象目标的压缩 Transformer 的三层引擎实现层级化时间动力学；以及通过图神经网络交互模式进行因果结构学习。HCLSM 引入了一种两阶段训练协议，其中空间重建在动力学预测开始前强制槽位专门化。我们在 Open X-Embodiment 数据集的 PushT 机器人操作基准上训练了一个拥有 6800 万参数的模型，实现了 0.008 MSE 的下一状态预测损失，并展现出涌现的空间分解（SBD 损失：0.0075）和学习到的事件边界。用于 SSM 扫描的自定义 Triton 内核相比顺序 PyTorch 实现了 38 倍的加速。整个系统包含分布在 51 个模块中的 8,478 行 Python 代码，并附带 171 个单元测试。代码：https://github.com/rightnow-ai/hclsm

## Abstract
World models that predict future states from video remain limited by flat latent representations that entangle objects, ignore causal structure, and collapse temporal dynamics into a single scale. We present HCLSM, a world model architecture that operates on three interconnected principles: object-centric decomposition via slot attention with spatial broadcast decoding, hierarchical temporal dynamics through a three-level engine combining selective state space models for continuous physics, sparse transformers for discrete events, and compressed transformers for abstract goals, and causal structure learning through graph neural network interaction patterns. HCLSM introduces a two-stage training protocol where spatial reconstruction forces slot specialization before dynamics prediction begins. We train a 68M-parameter model on the PushT robotic manipulation benchmark from the Open X-Embodiment dataset, achieving 0.008 MSE next-state prediction loss with emerging spatial decomposition (SBD loss: 0.0075) and learned event boundaries. A custom Triton kernel for the SSM scan delivers 38x speedup over sequential PyTorch. The full system spans 8,478 lines of Python across 51 modules with 171 unit tests. Code: https://github.com/rightnow-ai/hclsm

当前的世界模型（如 V-JEPA、DreamerV3）在处理视频预测时存在三大局限：
*   **物体纠缠（Entanglement）：** 采用扁平的潜空间（Flat Latent Space），将场景中的所有物体、背景和交互混合成一个无结构的向量，无法独立表征单个物体。
*   **时间尺度单一：** 物理世界的动态包含连续运动（如球滚动）、离散事件（如碰撞）和长期目标（如游戏策略），现有模型难以同时捕捉这些跨尺度的动态。

### 2. 方法论：核心思想与关键技术
HCLSM 提出了一个包含五个层级的复杂架构，并配合独特的两阶段训练协议：

*   **核心架构层级：**
    1.  **感知层（Perception）：** 使用 Vision Transformer (ViT) 将视频帧编码为补丁嵌入。
    2.  **物体分解层（Object Decomposition）：** 利用 **Slot Attention** 将场景分解为多个“槽位”（Slots）。引入**空间广播解码器（SBD）**，通过重建 ViT 特征（而非原始像素）来强制每个槽位专注于特定的空间区域。
    3.  **层级动力学层（Hierarchical Dynamics）：**
        *   **L0 级（SSM）：** 为每个物体分配独立的选择性状态空间模型（Selective SSM），处理连续的物理运动。
        *   **L1 级（Sparse Transformer）：** 仅在检测到“事件边界”（状态突变）时触发，处理离散交互。
        *   **L2 级（Goal Transformer）：** 通过交叉注意力压缩事件序列，进行高层抽象目标推理。
    4.  **因果推理层（Causal Reasoning）：** 通过图神经网络（GNN）学习物体间的交互，并利用 NOTEARS 约束来学习有向无环图（DAG）结构。
    5.  **持续记忆层（Continual Memory）：** 结合 Hopfield 网络和 EWC 算法处理长期记忆。

*   **两阶段训练协议（Two-Stage Training）：**
    *   **阶段 1（空间重建）：** 前 40% 的训练步数仅激活 SBD 重建损失，强制槽位实现空间专业化（即“先学会看什么是物体”）。
    *   **阶段 2（动力学预测）：** 激活 JEPA 风格的未来状态预测损失，在已解耦的物体基础上学习动态（即“再学会物体怎么动”）。
