## 摘要
我们提出了 SAL（SLAM 对抗实验室），这是一个用于在雾、雨等对抗条件下评估视觉 SLAM 系统的模块化框架。SAL 将每种对抗条件表示为一种扰动，将现有数据集转换为对抗性数据集。在转换数据集时，SAL 支持使用易于理解的现实世界单位（如以米为单位的雾天能见度）来表示严重程度。SAL 的可扩展架构通过通用接口实现了数据集、扰动和 SLAM 算法的解耦，使用户无需重写集成代码即可添加新组件。此外，SAL 还包含一个搜索程序，用于寻找导致 SLAM 系统失效的扰动严重程度。为了展示 SAL 的功能，我们的评估集成了七种 SLAM 算法，并在天气、摄像头和视频传输扰动下，跨三个数据集对它们进行了评估。

## Abstract
We present SAL (SLAM Adversarial Lab), a modular framework for evaluating visual SLAM systems under adversarial conditions such as fog and rain. SAL represents each adversarial condition as a perturbation that transforms an existing dataset into an adversarial dataset. When transforming a dataset, SAL supports severity levels using easily-interpretable real-world units such as meters for fog visibility. SAL's extensible architecture decouples datasets, perturbations, and SLAM algorithms through common interfaces, so users can add new components without rewriting integration code. Moreover, SAL includes a search procedure that finds the severity level of a perturbation at which a SLAM system fails. To showcase the capabilities of SAL, our evaluation integrates seven SLAM algorithms and evaluates them across three datasets under weather, camera, and video transport perturbations.

#### 1. 核心问题与整体含义（研究动机和背景）
视觉 SLAM 系统在理想基准测试中已达到高精度，但在实际部署中常面临恶劣天气（雨、雾）、相机退化（镜头污垢、裂纹、模糊）及视频传输异常（压缩、丢帧）等挑战。
*   **研究动机**：现有的鲁棒性评估方法存在三大痛点：
    1.  **缺乏可解释性**：扰动参数通常是抽象的（如“严重程度 3/5”），而非现实世界的物理单位。
    2.  **扩展性差**：扰动代码往往与特定数据集或算法强耦合，难以复用。
    3.  **难以定位失效边界**：无法自动确定算法从“正常工作”到“完全失效”的具体临界点。
*   **核心目标**：开发一个模块化、可扩展且具有物理意义的框架，用于系统性地评估视觉 SLAM 在对抗条件下的鲁棒性。

#### 2. 方法论：核心思想与关键技术
SAL 框架通过解耦设计实现了数据集、扰动模型和 SLAM 算法的独立集成。
*   **核心思想**：将对抗条件定义为“扰动模块”，通过通用接口对现有数据集进行变换，生成具有物理参数（如能见度米、降雨量 mm/h）的对抗样本。
*   **关键技术细节**：
    *   **深度感知扰动**：利用原生深度图或通过单目/双目深度估计模型（如 Depth Anything V2）生成深度信息，使雾气衰减、雨滴遮挡和运动模糊符合物理透视规律。

    *   **失效边界搜索（Boundary Search）**：引入基于**二分法（Bisection）**的搜索程序。用户设定一个性能阈值（如 ATE RMSE > 1.5m），系统自动在参数区间内迭代，高效定位算法崩溃的精确物理参数值。
    *   **复合扰动**：支持多种效应叠加（如“雨 + 镜头污垢”或“夜间 + 雾”），模拟复杂的真实环境。

#### 6. 主要结论与发现
*   **复合效应更具破坏性**：例如“夜间 + 雾”导致的误差远大于单一效应的叠加，部分算法在复合条件下误差激增数百倍。
*   **性能退化的突发性**：算法性能往往不是线性下降，而是在某个临界点突然崩溃（如 ORB-SLAM3 在能见度从 24m 降至 21m 时，误差从 0.3m 跃升至 28m）。
*   **算法敏感度差异**：传统特征法（ORB-SLAM3）在室外雨天相对稳定，但在极端低光下表现较差；深度学习方法（如 DROID-SLAM）在某些室内故障（如裂纹）下表现出更强的韧性。
*   **特征提取器的差异**：SuperPoint 在严重降雨和低带宽下比传统 ORB 特征更具生存能力。

#### 7. 优点
*   **物理可解释性**：将抽象的“噪声”转化为具体的“能见度”或“降雨量”，对工程部署具有直接指导意义。
*   **高度可扩展**：解耦设计使得研究人员可以轻松测试自己的算法或添加新的传感器失效模型。
*   **搜索效率高**：二分搜索显著降低了寻找算法鲁棒性边界的时间成本。

#### 8. 不足与局限
*   **模型简化**：运动模糊模型仅假设了纯前向运动，未考虑旋转或动态物体的复杂模糊。
*   **生成模型的局限**：昼转夜扰动依赖 CycleGAN-Turbo，虽然空间结构保持良好，但可能存在帧间的时间不一致性（闪烁）。
*   **深度依赖性**：对于缺乏原生深度的数据集，扰动质量受限于深度估计模型的准确性。
*   **覆盖范围**：尚未涵盖非视觉传感器（如 IMU 噪声、LiDAR 干扰）的对抗性评估。
